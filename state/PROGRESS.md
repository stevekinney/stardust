## 2026-07-07 ‚Äî Track C: Slash commands (complete)

Branch `track/slash-commands`, worktree `agent-a88270e1b6610665a`. Implements BUG-002 / the #1 OPENCLAW-GAPS.md gap: in-composer `/command` grammar.

- **Composer discovery**: expected a rich ProseMirror/Milkdown contenteditable (Cinder's `MarkdownEditor` supports a `wysiwyg` mode that renders that way) but confirmed via a live-DOM diagnostic test that Cinder's `Chat` composer actually renders as a plain `<textarea class="chat-input-editor" aria-label="Message">` ‚Äî this simplified the interception significantly (no ProseMirror transaction model to fight).
- **Interception is entirely Stardust-side**, in `src/lib/components/conversation-view.svelte`: a wrapper `<div>` around `<Chat>` registers `onkeydowncapture`/`oninputcapture` (Svelte 5 capture-phase event attributes). Capture phase runs before Cinder's own bubble-phase `onkeydown`/`onsubmit` handlers on the composer, so ArrowUp/ArrowDown/Enter/Escape are fully intercepted (`preventDefault`+`stopPropagation`) while the palette is open ‚Äî Cinder never sees them, so there's no risk of a stray submit. Typing `/` at the start of an empty composer (or any single-line, space-free text starting with `/`, to also cover paste/autofill) opens the palette; further typing narrows the fuzzy filter; a space, newline, or deleting past the `/` closes it.
- **Command logic lives in `src/lib/slash-commands.ts`** (pure, no DOM/Svelte) ‚Äî `createDefaultSlashCommands()` returns the six required commands (`/help`, `/new`, `/tools`, `/stop`, `/retry`, `/approvals`), each with an `unavailable(ctx)` check and a `run(ctx)` action returning `{ kind: 'close' }` or `{ kind: 'info', title, lines }`. `filterSlashCommands` does subsequence-based fuzzy matching against the command id, ranked by match tightness. `/stop` is disabled with "No run in progress to stop." when `!running`; `/retry` is disabled with "No previous turn to retry." when no `onRetry` handler is wired ‚Äî both wire to the _same_ durable signal paths the composer's Stop/Retry buttons already use (`onInterrupt`‚Üí`POST /api/sessions/[sessionKey]/interrupt`, `onRetry`‚Üíre-submitting `currentUserMessage`), not new plumbing. `/new` POSTs `/api/sessions` and navigates via `goto`/`resolve`, matching the existing pattern in `command-palette.svelte`. `/approvals` navigates to `/inbox`. `/help` and `/tools` render an inline "info" panel (`.slash-info`, a Stardust-owned block anchored above the composer, not injected into the durable transcript) rather than faking a chat message ‚Äî `/help` lists all commands, `/tools` calls the new `GET /api/tools` endpoint.
