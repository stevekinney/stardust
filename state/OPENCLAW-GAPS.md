# OpenClaw Parity Gaps

Source project researched: **OpenClaw** (formerly Clawdbot, then Moltbot — renamed after a trademark dispute with Anthropic), a self-hosted personal AI assistant. Repo: `github.com/openclaw/openclaw`, docs: `docs.openclaw.ai`. This is a real, actively maintained project — no substitution was needed. Chat surface researched: the OpenClaw "Control UI" web dashboard plus its shared slash-command layer. Sources are cited inline; all were fetched via web search/fetch by a research sub-agent and cross-checked against this repo's code by the discovery agent.

## Summary

Stardust is at parity or ahead of OpenClaw on tool-call visualization, streaming/typing indicators, interrupt/stop, error/retry UX, and ARIA live-region correctness (the latter specifically because Cinder already fixes the exact bug class OpenClaw has open). The two clear, unambiguous gaps are:

1. **No in-chat slash commands.** Stardust's ⌘K palette covers session/navigation actions but does not expose an in-composer `/command` grammar (e.g., `/retry`, `/interrupt`, `/model`) the way OpenClaw does. This is the single most actionable gap — Cinder's `Chat` composer doesn't currently define this, so it may need to be either a Stardust-side text-parsing layer over the existing `onSubmit` handler, or a Cinder feature request filed per this repo's own convention (`gh issue create --repo stevekinney/cinder`).
2. **No message edit + regenerate and no attachments.** Both are explicitly disabled via `capabilities={{ editing: false, attachments: false }}` in `src/lib/components/conversation-view.svelte:244-250` — Cinder's `Chat` component already supports both capability flags, so turning them on is a Stardust wiring decision (composer capability flag + a Stardust-side handler that resubmits from an edited point in the durable transcript), not a missing Cinder primitive.

## Update — 2026-07-07 (track/slash-commands)

Gap 1 ("No in-chat slash commands") above is **closed**. The current integration uses Cinder's public Chat composer API, introduced in 0.9 and retained by Stardust's current Cinder version: slash-command combobox state passes through the `composerRole` and `composerAria*` props, command keys are handled by `oncomposerkeydown`, and inline approval messages use `Chat.announce()`. The earlier composer reach-in has been removed. The minimum command set remains `/help`, `/new`, `/tools`, `/stop`, `/retry`, and `/approvals`—all wired to real durable-signal or API paths, not stubs. See `state/CINDER-RELEASE.md`, `state/PROGRESS.md`, `src/lib/slash-commands.ts`, `src/lib/components/slash-command-palette.svelte`, `src/lib/components/conversation-view.svelte`, and `src/routes/api/tools/+server.ts`.
