# Cinder Release — @lostgradient/cinder 0.9.0

## Current Stardust integration status

Stardust now targets `@lostgradient/cinder@0.9.0`. The 0.9 Chat composer API closes the remaining 0.8-era slash-command overlay gaps:

- `composerRole`, `composerAriaExpanded`, `composerAriaControls`, `composerAriaActiveDescendant`, and `composerAriaAutocomplete` carry slash-command combobox state through public `Chat` props.
- `oncomposerkeydown` handles ArrowUp/ArrowDown/Enter/Escape before Chat's own send shortcut, without wrapper capture handlers.
- `Chat.announce()` carries the inline approval announcement, so Stardust no longer needs a parallel app-owned live region for BUG-004.
- `conversation-view.svelte` no longer queries or detects `.chat-input-editor`.

## How it was published (actual mechanism)

Publishing is done by CI, not locally: `.github/workflows/release.yaml` on push to `main` with no pending changesets runs full `bun run validate` then `bun run --filter=@lostgradient/cinder publish:release -- --skip-validation` using **npm Trusted Publishing (OIDC)** — no npm token involved. The expired local npm token was therefore never a real blocker; the original local-publish attempt was simply the wrong path.

- Publishing run: [release run 28891301679](https://github.com/stevekinney/cinder/actions/runs/28891301679) — conclusion: success.
- An earlier release run (28889375601) failed once on a phantom test timeout and wedged once on rerun — diagnosed as chronic runner/module-init flakiness in that repo's CI (pre-existing, our commit's own main-green run passed). Full evidence filed as [cinder#683](https://github.com/stevekinney/cinder/issues/683).
