### Dev-server start command (used for the live walkthrough)

`.claude/launch.json` defines a `web` configuration: `bun run dev` on port 7777 (autoPort). This is `scripts/dev.ts`, a one-command orchestrator that:

1. Checks for a reachable Temporal dev server at the configured `TEMPORAL_ADDRESS`. In this environment `.env` pointed at a Temporal Cloud address; the script correctly detected it wasn't reachable locally, **dropped the leaked cloud API key**, and started a local dev server on `localhost:7234` (UI `8234`) instead — auto-selecting a free port because `7233`/`8233` were already occupied by another unrelated Temporal instance running on this machine.
2. Applies DB migrations (`tsx scripts/db/migrate.ts`).
3. Boots the SvelteKit web process (`vite dev`, port 7777) and the Temporal Worker (`tsx watch src/worker/main.ts`) concurrently via `concurrently`.

Confirmed via `mcp__Claude_Preview__preview_start` (`name: "web"`) — the app came up cleanly on `http://localhost:7777`, five Temporal task-queue workers registered (`agent-orchestrator`, `model-calls`, `tools-general`, `tools-sandbox`, `memory`), and a full chat turn (send → stream → tool call → approval → completion) was exercised live (see `state/UX-AUDIT.md`).

E2E (`bun run test:e2e`) was **not** run in this pass — it installs Playwright browsers and runs a full suite; deferred to avoid a long, resource-heavy run during discovery. Typecheck/lint/unit/build are the load-bearing gates and all passed clean, which is the acceptance bar for this phase.
